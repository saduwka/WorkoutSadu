# SADU

iOS‑приложение **SADU**: тренировки, здоровье, задачи, финансы, привычки. SwiftUI, SwiftData. Виджеты, Share Extension для чеков, ИИ‑чат (Gym Bro / Life Bro).

---

**Update 05.03.2026**

### Share Extension (чеки в «Деньги»)

- **Правило активации** — расширение показывается только для PDF и изображений: в `Info.plist` задан предикат для типов `com.adobe.pdf`, `public.image`, `public.jpeg`, `public.png`.
- **Darwin‑уведомление** — после сохранения чека (подтверждённые транзакции или отложенный текст) расширение шлёт Darwin‑нотификацию `com.saduwka.WorkoutSadu.receiptSaved`. Основное приложение подписано в `AppDelegate` и по ней вызывает `WidgetCenter.shared.reloadAllTimelines()` для обновления виджетов.

### Финансы

1. **Переводы между счетами** — кнопка «Перевод между счетами» в обзоре (при 2+ счетах), экран выбора «откуда» / «куда», сумма и опциональная комиссия. Создаются расход и доход + при необходимости комиссия (категория «Переводы»).
2. **Переводы не в общий доход/расход** — операции с категорией «Переводы» везде исключены из итогов «доходы» и «расходы» (обзор, сегодня, статистика, история, сводка для виджета).
3. **Редактирование транзакций** — тап по строке в истории открывает лист редактирования (название, сумма, категория, тип, дата, счёт); удаление — через контекстное меню (долгое нажатие).

### Share Extension (чеки)

4. Доработка сценария — после распознавания чека показывается «Добавить в Деньги?» с парсингом (SimpleReceiptParser), кнопки «Добавить» и «Нет, в приложении».
5. Экран «Добавить в Деньги?» — выбор категории расхода и редактирование описания (имя).
6. Окно «успешно» — после «Добавить» экран успеха, затем закрытие через 2 сек.
7. Окно при «Нет, в приложении» — текст «Текст чека сохранён» и инструкция открыть приложение → «Финансы», закрытие через 2,5 сек.

### Life Bro (ИИ)

8. Полные данные в контексте — при запросах вроде «что тратил», «покажи тренировки», «какие задачи» ИИ видит: последние транзакции, счета и балансы, последние тренировки, список невыполненных задач (в системном промпте).

### Инфраструктура и бэкап

9. `.cursorrules` — правила стиля и контекста проекта (язык, дизайн, SwiftUI/SwiftData, финансы, Share Extension).
10. Автобэкап в Firebase — раз в 24 часа при запуске/возврате данные SwiftData экспортируются в Firebase Storage (`backups/{uid}/latest.json`). FirebaseAuth (анонимный вход), FirebaseStorage, сервис `FirebaseBackupService`, триггер в `WidgetSyncModifier`. Подробнее — `BACKUP_FIREBASE.md`.

### Подсказка по запуску

11. **Виджет** — отдельной схемы нет; выбирать схему основного приложения WorkoutSadu и запускать её — виджет ставится вместе с приложением.
