Update 03.03.2026
 Финансы

  1. Переводы между счетами — кнопка «Перевод между счетами» в обзоре (при 2+ счетах), экран выбора «откуда» / «куда», сумма и опциональная комиссия. Создаются расход и доход + при необходимости
     комиссия (категория «Переводы»).
  2. Переводы не в общий доход/расход — операции с категорией «Переводы» везде исключены из итогов «доходы» и «расходы» (обзор, сегодня, статистика, история, сводка для виджета).
  3. Редактирование транзакций из истории — тап по строке открывает лист редактирования (название, сумма, категория, тип, дата, счёт); удаление — через контекстное меню (долгое нажатие).


  ────────────────────────────────────────



  Share Extension (чеки в «Деньги»)

  4. Доработка сценария — после распознавания чека показывается «Добавить в Деньги?» с парсингом (SimpleReceiptParser), кнопки «Добавить» и «Нет, в приложении».
  5. Исправление компиляции — один общий тип ShareStep, фон кнопки через UIColor.clear.
  6. Экран «Добавить в Деньги?» — выбор категории расхода и редактирование описания (имя).
  7. Окно «успешно» — после «Добавить» показывается экран успеха, затем закрытие через 2 сек.
  8. Окно при «Нет, в приложении» — экран с текстом «Текст чека сохранён» и инструкцией открыть приложение → «Финансы», закрытие через 2,5 сек.


  ────────────────────────────────────────



  Life Bro (ИИ)

  9. Полные данные в контексте — при запросах вроде «что тратил», «покажи тренировки», «какие задачи» ИИ видит: последние транзакции, счета и балансы, последние тренировки, список невыполненных задач
      (в системном промпте).


  ────────────────────────────────────────



  Инфраструктура и бэкап

  10. `.cursorrules` — файл правил стиля и контекста проекта (язык, дизайн, SwiftUI/SwiftData, финансы, Share Extension).
  11. Автобэкап в Firebase — раз в 24 часа при запуске/возврате в приложение данные SwiftData экспортируются в Firebase Storage (backups/{uid}/latest.json). Добавлены FirebaseAuth (анонимный вход) и
      FirebaseStorage, сервис FirebaseBackupService, триггер в WidgetSyncModifier, инструкция в BACKUP_FIREBASE.md (что включить в Firebase Console).


  ────────────────────────────────────────



  Подсказка по запуску

  12. Запуск виджета — виджет не запускается отдельной схемой; нужно выбирать схему основного приложения WorkoutSadu и запускать её — виджет ставится вместе с приложением.
