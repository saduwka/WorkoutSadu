## [3.15] — 2026-03-15

### Добавлено

- **Еда** — добавление еды за выбранный день: при открытии «Добавить еду» из раздела «Еда» подставляется дата выбранного дня (`AddMealView(initialDate:)`), можно добавить приём за вчера или любой другой день.
- **Кардио** — гибкая схема метрик из JSON: конфиг по имени упражнения (Степпер, Беговая дорожка, Велотренажёр, Гребной и т.д.) или из `CardioPresets.json`; хранение в `WorkoutExercise.cardioValuesJson`; блок «КАРДИО» в экране упражнения строится по конфигу (время, расстояние, шаги/мин, ступени, мощность, ЧСС и др.).
- **Life Bro** — в промпт шаблонов добавлено правило по кардио: использовать названия тренажёров, чтобы в приложении подставлялась нужная форма ввода.
- **Отчёты** — по тапу на push «Итоги дня» / «Недельный» / «Месячный» открывается соответствующий экран отчёта с авто-запросом комментария Life Bro.
- **Дневной отчёт** — приёмы пищи (что и во сколько) и категории расходов в снимке; промпт требует от Life Bro обязательный блок по еде (что ел, во сколько, совет) и краткий блок по финансам; то же по финансам для недельного и месячного промптов.
- **История уведомлений** — уведомления отчётов отображаются в окне по колокольчику; по тапу открывается полный отчёт за тот же период.

### Изменено

- **Главный экран** — заголовок навбара заменён с «LIFEOS» на «SADU»; в промпте Life Bro приложение указано как SADU.
- **Share Extension (чеки)** — убрано второе окно с OCR в расширении. После «Это оплата?» → «Да» расширение сохраняет файл (PDF/фото) в App Group и показывает «Чек передан в приложение» / «Откройте приложение и продолжите в разделе «Деньги»», затем закрывается. Распознавание и парсинг выполняются в основном приложении при открытии вкладки «Деньги».

### Исправлено

- **Secrets.plist** — удалена ошибочная строка без ключа (ошибка property list).
- **NotificationDelegate** — восстановлено свойство `modelContext` для сохранения уведомлений в историю.
- **Отчёты** — снимки дня/недели/месяца приведены к `Equatable` для `onChange(of: snapshot)` (Swift 6).
- **Swift 6 / iOS** — `GymBroMessage.Role`: добавлен `RawValue`, убрано использование Equatable в nonisolated контексте; замена deprecated `+` для `Text` на интерполяцию (iOS 26); `@preconcurrency import Vision`, создание `handler`/`request` Vision внутри очереди (Sendable); `AVAudioApplication.requestRecordPermission()`; неиспользуемый результат `try?` и прочие мелкие правки.
