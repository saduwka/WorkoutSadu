# Автобэкап в Firebase

Раз в 24 часа (при открытии приложения или возврате в него) данные SwiftData экспортируются в Firebase Storage.

## Что уже сделано в коде

- Добавлены пакеты **FirebaseAuth** и **FirebaseStorage** в таргет приложения.
- Сервис `FirebaseBackupService`: анонимный вход, сбор данных (тренировки, финансы, привычки, задачи, цели, профиль, питание) в JSON, загрузка в Storage.
- Триггер: при запуске приложения и при переходе в foreground вызывается `tryExportIfNeeded`; если с последнего экспорта прошло ≥24 ч — выполняется экспорт.

## Что нужно сделать в Firebase

1. **Authentication**
   - В [Firebase Console](https://console.firebase.google.com) → твой проект → **Authentication** → **Sign-in method**.
   - Включи **Anonymous** (анонимный вход). Этого достаточно: для бэкапа создаётся анонимный пользователь, по его `uid` кладётся файл.

2. **Storage**
   - **Storage** → **Rules**.
   - Разреши запись в папку бэкапов только для авторизованных пользователей в своём сегменте, например:
   ```
   rules_version = '2';
   service firebase.storage {
     match /b/{bucket}/o {
       match /backups/{userId}/{allPaths=**} {
         allow read, write: if request.auth != null && request.auth.uid == userId;
       }
     }
   }
   ```
   Так пользователь сможет писать только в `backups/<свой uid>/`.

3. **Путь в Storage**
   - Файл сохраняется по пути: `backups/{uid}/latest.json`.
   - Один файл на устройство (анонимный пользователь = один аккаунт на приложение). При переустановке будет новый анонимный пользователь и новый файл.

## Восстановление (на будущее)

Сейчас реализован только экспорт. Импорт из `latest.json` можно добавить отдельно: скачать файл из Storage, распарсить JSON и заполнить SwiftData (с конфликтами по id на твоё усмотрение).
